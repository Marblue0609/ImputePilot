# ImputePilot Project Memory

## Scope and Entry Points
- Main Django project root: `ImputePilot/demo`.
- Start server from `ImputePilot/demo` with `python manage.py runserver`.
- Django settings module: `ImputePilot/demo/demo/settings.py`.
- API base route: `/api/` from `ImputePilot/demo/demo/urls.py`.

## Backend Structure
- Core web app: `ImputePilot/demo/ImputePilot_api`.
- Primary API implementation is concentrated in `ImputePilot/demo/ImputePilot_api/views.py` (large monolithic file).
- URL mapping for dashboard/pipeline/baseline/recommend/evaluation is in `ImputePilot/demo/ImputePilot_api/urls.py`.
- Celery task for async feature extraction: `ImputePilot/demo/ImputePilot_api/tasks.py`.

## Algorithm/Core Logic (RecImpute-derived)
- Main package: `ImputePilot/demo/ImputePilot_api/ImputePilot_code`.
- Clustering: `Clustering/ShapeBasedClustering.py` + `ConFree_kClustering.py`.
- Labeling: `Labeling/ImputationTechniques/ImputeBenchLabeler.py`.
- Feature extraction: `FeaturesExtraction/` (Catch22, TSFresh, Topological, etc.).
- Training/selection: `Training/ClfPipeline.py`, `Training/ModelsTrainer.py`, `Training/RecommendationModel.py`, `Datasets/TrainingSet.py`.
- Config files: `ImputePilot/demo/ImputePilot_api/ImputePilot_code/Config/*.yaml`.

## Frontend Structure
- Template: `ImputePilot/demo/ImputePilot_api/templates/ImputePilot_api/ImputePilot_v3.html`.
- Frontend logic: `ImputePilot/demo/ImputePilot_api/static/ImputePilot_api/app.js`.
- Styling: `ImputePilot/demo/ImputePilot_api/static/ImputePilot_api/style.css`.

## External Baseline Runners
- Located at: `ImputePilot/demo/baseline_runners/`.
- Separate scripts for FLAML, Tune, RAHA; communicate through npz/json contract.
- Runtime state artifacts stored in `ImputePilot/demo/baseline_runners/.state/`.
- Env vars wired in settings: `BASELINE_RUNNERS_DIR`, `FLAML_VENV_PY`, `TUNE_VENV_PY`, `RAHA_VENV_PY`.

## External DL Labeling/Imputation Integration
- Runner entry: `ImputePilot/demo/labeling_runners/dl_benchmark_runner.py` (mode: `benchmark` for labeling, `impute` for inference).
- Settings/env: `DL_LABEL_ENV_NAME`, `DL_LABEL_VENV_PY`, `DL_LABEL_RUNNER_SCRIPT`, `DL_LABEL_ALGO`, `DL_LABEL_DEVICE`, `DL_LABEL_CELERY_QUEUE`, `DL_IMPUTE_ALGOS`, `DL_LABEL_TIMEOUT_SEC`.
- Current DL algos wired: BRITS (PyPOTS) + DeepMVI/MRNN/MPIN/IIM (ImputeGAP). Extend by adding to `DL_IMPUTE_ALGOS`, frontend `EXTERNAL_DL_ALGOS`, and runner dispatch.
- Frontend: `ImputePilot/demo/ImputePilot_api/static/ImputePilot_api/app.js` auto-enables external runner when a DL algo is selected; hides the extra confirmation toggle.
- Labeling override: `ImputeBenchLabeler` supports `<dataset>_override_labels.csv` (generated by `views.run_labeling`) to override per-cluster labels; cleared when external DL not used.
- Imputation: `views.run_imputation` routes external DL algos to `_run_external_imputation_for_algo`, writes `imputed_<algo>_*.csv` to inference dir, and computes RMSE/MAE if ground truth/mask exists.
- External DL labeling can evaluate multiple DL algos per cluster via Celery tasks and choose the best RMSE on a shared mask.

## External DL Environment (recommended default)
- Conda env: `ImputePilot-labeldl311` (Python 3.11).
- Minimal installs (CPU): `torch==2.2.2+cpu` (PyTorch CPU index), `numpy==1.26.4`, `h5py`, `pygrinder`, `toml`, plus `pypots==1.1 --no-deps` and `imputegap==1.1.1 --no-deps`.
- GPU mode (optional): `torch==2.2.2+cu121` (PyTorch CUDA index), enable with `DL_LABEL_DEVICE=auto` and a Celery worker on `DL_LABEL_CELERY_QUEUE` (default `dl_gpu`).
- MRNN requires `tensorflow==2.16.2` in `ImputePilot-labeldl311`.
- Rationale: PyPOTS BRITS import is direct (`pypots.imputation.brits`) to avoid heavy optional deps; DeepMVI uses ImputeGAP implementation.
- Local patches in the env to keep imports light:
  - `pypots/__init__.py` and `pypots/imputation/__init__.py` trimmed to avoid importing all submodules.
  - `imputegap/wrapper/AlgoPython/DeepMVI/recovery.py` moves `Imputation`/`TimeSeries` imports into `__main__`.

## Data/Artifacts Paths Used by API
- Training datasets: `ImputePilot/demo/ImputePilot_api/ImputePilot_code/Datasets/RealWorld`.
- Inference datasets: `ImputePilot/demo/ImputePilot_api/ImputePilot_code/Datasets/Inference`.
- System inputs: `ImputePilot/demo/ImputePilot_api/ImputePilot_code/Datasets/SystemInputs`.
- Recommendation outputs: `ImputePilot/demo/ImputePilot_api/ImputePilot_code/Datasets/Recommendations`.

## Current High-Level Workflow in API
1. Upload training data.
2. Run clustering.
3. Run labeling.
4. Run features.
5. Run model race and cache trained model in `AdartsService`.
6. Upload inference/evaluation data.
7. Get recommendation + optional baseline comparison/imputation/downstream evaluation.

## Noted Integration Risks (to verify when fixing bugs)
- Frontend calls evaluation metrics with GET while backend endpoint requires POST.
- `setup_upload` frontend flow sends JSON, but backend endpoint currently expects uploaded files in `request.FILES`.
- `compute_ground_truth_labels` builds full file paths then joins with inference dir again, which can produce invalid paths.
